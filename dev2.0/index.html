스마트 플랫폼 구축 기획안

(v2.0)

사용자 유형과 접속 주소에 따라 개인화된 UI(상품, 로고, 메뉴)를 제공하는 **동적 UI 시스템** 구축을 위한 최종 통합 기획안입니다.

1. 기획 요약 (Executive Summary)

프로젝트 목표: 사용자의 유형을 **일반 회원**과 **전용 회원**으로 나누어, 각기 다른 상품 탐색 경험과 맞춤형 UI를 제공하는 지능형 식자재 유통 플랫폼을 구축합니다.

핵심 컨셉: 두 종류의 회원

① 일반 회원 (GENERAL)

**본사의 마케팅 활동을 통해 유입**되며, 자신의 위치를 기반으로 배송 가능한 유통사와 상품을 자유롭게 탐색하고 거래할 유통사를 직접 선택합니다.

  • **유통사 탐색:** 지역, 상품 등으로 최적의 유통사 검색

② 전용 회원 (DEDICATED)

특정 유통사와의 파트너십을 통해 독점 상품, 계약 단가 등 특별한 혜택을 제공받습니다.

  • **독점적 관계:** 지정된 유통사 상품만 집중 노출
  • **혜택 극대화:** 계약 단가 및 여신(외상) 한도 적용

개발 전략: 가장 핵심적인 **'회원 유형에 따른 동적 상품 노출' 기능을 MVP(Minimum Viable Product)로 우선 개발**한 후, UI 브랜딩 기능을 점진적으로 확장합니다.

2. Phase 1: MVP 개발 - 동적 상품 노출

시스템의 기반을 다지고, 가장 핵심적인 '상황에 맞는 상품 노출' 기능을 구현하여 빠르게 가치를 검증합니다.

STEP 1: 인프라 설정 (Infra Setup)

와일드카드 DNS 설정: 도메인 관리 시스템에서 `*.orderz.kr` 레코드가 서버 IP를 가리키도록 설정합니다. (전용 회원용)

STEP 2: 백엔드 개발 (Back-end)

DB 스키마 확장:
  • `Users` 테이블: `member_type` ('GENERAL'/'DEDICATED'), `distributor_code` (전용회원용), `address` (일반회원용) 컬럼 추가
  • `Distributors` 테이블: `delivery_areas` (배송가능지역) 컬럼 추가
  • `Products` 테이블: `distributor_code` (상품소속) 컬럼 추가
맞춤형 상품 API 개발: 아래 프로세스 흐름도에 따라 상품 목록을 결정하는 API를 구현합니다.

3. Phase 2: 기능 확장 - UI 개인화 (전용 회원용)

MVP 기능 위에 로고, GNB 메뉴 등 UI 브랜딩 기능을 추가하여 **전용 회원**에게 완전한 '전용몰' 경험을 제공합니다.

STEP 1: 관리자 기능 개발 (Admin)

로고 관리: 유통사가 전용 로고 이미지를 업로드하는 기능 구현
GNB 관리: 유통사가 GNB 메뉴의 '명칭'과 '링크'를 설정하는 기능 구현

STEP 2: 동적 UI 적용 (Back-end & Front-end)

상품 API 응답에 `logoUrl`, `gnbInfo` 등 추가 데이터를 포함하도록 백엔드 수정
프론트엔드에서 API 데이터를 받아 헤더의 로고와 GNB를 동적으로 변경
